# Save Words · 雅思单词抢答小游戏

> 🎮 把枯燥的雅思 538 考点词，变成一场"在 10 秒内拯救单词"的抢答游戏。

一个极简的单文件网页小游戏。单词会像俄罗斯方块一样从屏幕上方掉下来，你要在它"落地"之前，迅速选出**所有**同义词，按下确认键把它救回来。



---

## ✨ 这个游戏为什么存在？

背雅思单词太无聊了。

《雅思真经·阅读考点词 538》是本好书，但一页一页地啃下去实在需要毅力。更关键的是——**看完了，大脑在考场上也未必能瞬间联想到它的同义替换**，而这恰恰是雅思阅读的核心考点。

于是我想：既然大脑天生讨厌枯燥的记忆，那干脆把它**游戏化**。

我不是程序员，用 AI 工具一点点"搭"出了这个叫 **Save Words** 的小游戏。它的设计哲学很简单：

### 🎯 精准拿捏雅思内核
雅思阅读的核心是"同义替换"。所以这个游戏**完全不考拼写**，只考你**对同义词的反应速度**。和 538 考点词的精髓完美契合。

### 🙅 拒绝假装努力
看书是"被动输入"，玩游戏是"主动检索"。当进度条一点点掉下来的时候，脑子真的会转得飞快。

### ⚡ 那种微妙的紧张感
看着单词一格一格下坠，手会不自觉地抖。为了不让单词"挂掉"，大脑会强迫自己进入一种极度专注的检索状态。这种压力恰到好处——比死记硬背强 10 倍。

### 🔔 拍灯的爽感
确认键被刻意做得很大，点下去的那一下，像极了抢答按铃。反馈感拉满。

## 🎮 游戏规则

```
20 回合  ·  3 条命  ·  每个单词约 11 秒
```

| 元素 | 说明 |
|---|---|
| ⬇️ 下落单词 | 屏幕左侧顶部会掉下一个英文单词（带简短中文提示） |
| ☑️ 右侧选项 | 屏幕右侧列出 6 个候选词，其中包含**多个正确同义词** |
| ✅ 胜利条件 | 在单词"落地"前，选中**所有**正确选项（子集必须完全匹配），再点 CONFIRM |
| ❌ 失败条件 | 时间到了没选对，或者选错/漏选，扣一条命 |
| 💀 Game Over | 3 条命用完，游戏结束 |
| 🏆 通关 | 成功救下 20 个单词 |

### 操作方式
- **鼠标 / 触屏**：点击选项卡片切换选中状态，点 `CONFIRM` 提交
- 支持手机浏览器直接打开（响应式布局）

## 🧠 为什么这样设计？

在折腾这个小游戏的过程中，我一直在想一个问题：

> **为什么我们能打几小时游戏不累，却很难背半小时单词？**

答案其实很简单——学习本身不该是"被动受罪"。大脑真正需要的是：

1. **即时反馈**（对/错立刻告诉我）
2. **清晰目标**（救下这一个单词就行）
3. **微任务化**（不是"背完 538 页"，而是"10 秒内做一题"）
4. **适度压力**（进度条制造的紧张感，让专注成为本能）

当"背单词"这个沉重的包袱，被拆解成一个个"在 10 秒内拯救一个单词"的微任务时，**那种对抗枯燥的阻力感就消失了**。

我们不需要战胜人性去学习——我们需要的是**顺应人性的工具**。

## 🚀 快速开始

### 方式 1：直接打开（最简单）
下载 `index.html`，双击用浏览器打开即可。无需任何安装。

### 方式 2：本地预览
```bash
git clone https://github.com/<your-username>/save-words.git
cd save-words

# 任选一种方式起静态服务器
python3 -m http.server 8000
# 或
npx serve .
```
然后浏览器访问 `http://localhost:8000`。

### 方式 3：GitHub Pages 一键部署
1. Fork 本仓库
2. 进入仓库 `Settings` → `Pages`
3. `Branch` 选 `main`，目录选 `/ (root)`，保存
4. 几十秒后即可通过 `https://<你的用户名>.github.io/save-words/` 访问

## 🛠 技术栈

刻意保持极简——**单文件、零构建、零依赖安装**。

- **HTML + CSS**：纯手写，没有 UI 框架
- **React 18 UMD**：通过 CDN 直接引入，不用 npm / webpack / vite
- **Web Audio API**：生成按键音效（不依赖任何音频文件）
- **Google Fonts**：`Space Mono` + `Noto Serif SC`（赛博朋克 × 中式书卷感）

整个游戏只有**一个 `index.html` 文件**，方便你随便改、随便部署。

- **UI / 交互有想法** → 直接 PR 或 Issue 讨论

### 词条数据格式

在 `index.html` 中找到 `const WORDS=[...]`，每一条长这样：

```js
{
  word: 'stem from',                    // 要背的词
  cn: '起源于（stem from）',             // 简短中文提示
  correct: [                            // 所有正确的同义替换词
    'derive',
    'originate',
    'come from',
    'result from'
  ]
}
```

干扰项（错误选项）会从**其他词条的 correct 列表里**随机抽取，所以你贡献的词条越多，干扰越真实。
